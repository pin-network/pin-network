How it works

PiN is a mesh network. Every node is equal. Every node earns.

1

Flash or install

Flash the PiN image to a Raspberry Pi, or install the tray app on any device. One step. No configuration needed.

2

Join the mesh

Your node finds peers automatically using a distributed hash table. No central server. No signup. No account.

3

Serve and earn

Your node serves web content to other users. Every byte you serve earns Hashes — the PiN proof-of-service token.

4

Host anything

Publish your own .pin site from the browser. Hashes can boost your site's priority or be shared with others.

Node tiers

Every device is welcome. You set the limits. You choose the hours.

TIER 1

Raspberry Pi nodes

RPI 3, 4, 5 · Zero 2W · Any SBC
  • Always on, low power
  • Static hosting, webhooks
  • File and asset delivery
  • Base Hash rate
TIER 2

Soft nodes

Phone · Tablet · Laptop
  • Opt-in with schedule control
  • JSON APIs, caching, relay
  • Battery and WiFi rules
  • 2× Hash rate
TIER 3

Power nodes

Desktop · Mini PC · NAS · Server
  • Maximum availability
  • Heavy compute and large storage
  • Dynamic apps and databases
  • 4× Hash rate

Earn Hashes for real work

PiN uses proof-of-service, not proof-of-work. Hashes are not mined by solving puzzles — they are earned by serving content to real users. The more you contribute, the more you earn.

Hashes = bytes_served × uptime × tier_multiplier

Hashes boost your hosted content's priority, purchase additional bandwidth, or can be shared with other contributors. The ledger is fully distributed — no central authority controls it.

Connect from anywhere

PiN is built for the real world — including places with unreliable or expensive internet.

WiFi & Ethernet

Primary mode for home and office nodes. A 5 Mbps upload is more than enough for Tier 1 hosting.

Cellular

Works with any existing data plan. PiN respects your data by applying conservative defaults on cellular connections.

Point-to-point

Directional WiFi bridges for line-of-sight rural links. Standard network interface — no special configuration.

LoRa / off-grid

Coming Phase 4. Low-bandwidth mesh for remote environments using Meshtastic-compatible hardware.
